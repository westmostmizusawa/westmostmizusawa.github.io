Hexo+Github 搭建个人简易博客
Created 2020-06-02| Updated 2020-07-16|record skill
Post View:

一、Hero简介
Hexo官网：https://hexo.io/zh-cn/

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

其实说白了就是将我们所熟悉的写博客文件的格式.md解析成了html，服务数据什么的官方已经帮助解决，我们再将其托管在GitHub Pages上，这样就可以实现完全免费构建博客，当然推荐爬梯，否则会很慢

二、安装与配置
适用系统：windows
1.安装Nodejs
这个是必须的，因为hexo就是基于nodejs的，从官网下载压缩包，安装即可

    （1）Node.js 安装包及源码下载地址为：https://nodejs.org/en/download/
        选择Windows 安装包(.msi)，根据情况选择32位和64位，按默认向导安装。
    （2）检测PATH环境是否配置Node.js，在cmd中，输入命令 path ，显示如下
        C:\Program Files\nodejs\;
        C:\Users\rg\AppData\Roaming\npm
        我们可以看到环境变量中已经包含了C:\Program Files\nodejs\
        可以通过 node --version 检查Node.js版本
    （3）不过这里的命令不是全局的，所以需要把node和npm设置成全局
        ln -s /home/node-v4.4.4-linux-x64/bin/node /usr/local/bin/node  
        ln -s /home/node-v4.4.4-linux-x64/bin/npm /usr/local/bin/npm
        前边的/home/node-v4.4.4-linux-x64/bin/npm 是自己的node所在的路径，这个记得改成自己的
2.安装git
    我的电脑上已经有了，这里就不多说了
3.安装hexo
    在GitBash中运行
        npm install hexo-cli -g

三、建站
1.初始化
hexo安装完之后，需要初始化一个博客的目录(ps:初始化目录可以自行创建也可以以下面这种方式创建)：
$ hexo init folder
（注 这里的folder表示hexo初始化在此文件夹目录下）
2. 安装依赖包
进入刚刚初始化目录，安装…
$ cd
$ npm install
安装完成之后目录结构是这样的：
.
├── _config.yml // 整个网站信息的配置文件
├── package.json // 依赖包文件的配置
├── scaffolds // 新建文章时通过它建立文件
├── source // 原文件目录
| ├── _drafts
| └── _posts
└── themes // 主题目录
有的人安装完之后没有_drafts，没关系的，没有会把我们的所有源文件解析放在 public 文件夹下
3. 启动服务
$ hexo server 或者 $ hexo s
看到INFO Start processing INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.即为成功，直接访问即可：
四、配置hexo页面主题
修改初始化的文件夹下的 _config.yml 文件

五、Github托管

1.在自己的Github上创建一个新仓库，仓库名如下
    你的GitHub账户名.github.io
2.再次打开博客根目录下的_config.yml，找到 deploy 字段相应的设置
    deploy:
    type: git
    repository: github地址
    branch: master
3.修改完_config.yml后，保存
    其实就是给hexo d 这个命令做相应的配置，让hexo知道你要把blog部署在哪个位置，很显然，我们部署在我们GitHub的仓库里。最后安装Git部署插件，输入命令：
    $ npm install hexo-deployer-git --save
    这时，我们分别输入三条命令：
    $ hexo clean 
    $ hexo g 
    $ hexo d
    其实第三条的 hexo d 就是部署网站命令，d是deploy的缩写。
4.部署到Github上
    $ hexo d
这样就可以把博客部署到github上了，比如我的博客地址是https://westmostmizusawa.github.io/

六、博客编写
1.发表文章，过程依次参照下面几步：

    $ hexo new <title>      // title 就是要发表的文章名
    $ hexo generate 或者 $ hexo g        // 生成静态文件
    $ hexo server          // 启动服务

2.
    hexo就那么几个常用的命令：
    hexo n 新建博客
    hexo g 生成博客
    hexo d 部署博客
    hexo s 开启服务

八、杂项
文章可以拥有如下属性：
- Setting Description Default
1.layout Layout post或page
2.title 文章的标题
3.date 创建日期 文件的创建日期
4.updated 修改日期 文件的修改日期
5.comments 是否开启评论 true
6.tags 标签
7.categories 分类
8.permalink url中的名字 文件名
动态博客中通过发布文章页面设置的各种属性，在hexo里要这样设置。
